# Dev Fullstack Template

[![中文文档](https://img.shields.io/badge/README-中文文档-blue)](README.zh-CN.md)

Python 3.12 + FastAPI + Next.js fullstack development template.

This template borrows the engineering layout from Dify:

- root-level orchestration through `Makefile`
- Dify-style thin scripts under `dev/`
- isolated backend project under `api/`
- isolated frontend project under `web/`
- Docker middleware and full-stack compose files under `docker/`
- `.env.example` files as configuration templates
- devcontainer support with Python 3.12, Node.js, pnpm, uv, and Docker-in-Docker

## Project Layout

```text
.
├── api/                    # FastAPI backend, managed by uv
├── web/                    # Next.js frontend, managed by pnpm
├── dev/                    # Daily development scripts
├── docker/                 # Docker compose, env templates, env sync
├── .devcontainer/          # Dev Container definition
├── Makefile                # Common project commands
└── README.md
```

## Quick Start

```bash
make dev-setup
make docker-up
make dev
```

Services:

- Web: http://localhost:3000
- API: http://localhost:5001
- API docs: http://localhost:5001/docs
- Postgres: localhost:5432
- Redis: localhost:6379

## Daily Development

Use the root `Makefile` for common tasks:

```bash
make dev-setup      # Create env files and install dependencies
make docker-up      # Start local middleware: Postgres and Redis
make docker-down    # Stop local middleware
make dev            # Start API and Web dev servers
make check          # Run backend/frontend lint and type checks
make test           # Run backend/frontend tests
make format         # Format and auto-fix code
make build-api      # Build backend Docker image
make build-web      # Build frontend Docker image
make compose-up     # Start full Docker stack
make compose-down   # Stop full Docker stack
```

The `dev/` directory exposes the same workflow as explicit scripts:

```bash
dev/setup
dev/start-docker-compose
dev/stop-docker-compose
dev/start-api
dev/start-web
dev/check
dev/test
dev/reformat
```

## Backend

The backend lives in `api/`.

Stack:

- Python 3.12
- FastAPI
- uv
- Ruff
- mypy

Run only the backend:

```bash

```

Useful backend commands:

```bash
cd api
uv sync --dev
uv run uvicorn app.main:app --host 0.0.0.0 --port 5001 --reload
uv run ruff check .
uv run mypy app tests
uv run pytest
```

## Docker Layout

Docker configuration lives in `docker/`.

```text
docker/
├── compose.middleware.yaml     # Local development middleware only
├── compose.template.yaml       # Source template for full Docker stack
├── compose.yaml                # Generated full Docker stack
├── generate-compose            # Regenerates compose.yaml
├── env-sync.py                 # Syncs .env files from .env.example files
├── env-sync.sh                 # Shell wrapper for env-sync.py
└── envs/
    ├── core-services/
    └── databases/
```

`compose.middleware.yaml` is for local development. It starts only Postgres and Redis.

`compose.template.yaml` is the source file for the complete Docker stack.

`compose.yaml` is generated. Do not edit it directly.

Regenerate it with:

```bash
docker/generate-compose
```

Start local middleware:

```bash
make docker-up
```

Start the full Docker stack:

```bash
make compose-up
```

## Environment Files

Each `.env.example` file is a template. Local `.env` files are generated by `dev/setup` or `docker/env-sync.sh`.

Examples:

```text
api/.env.example                       -> api/.env
web/.env.example                       -> web/.env.local
docker/.env.example                    -> docker/.env
docker/envs/core-services/api.env.example
                                      -> docker/envs/core-services/api.env
```

Sync Docker env files after template updates:

```bash
docker/env-sync.sh
```

The sync tool:

- creates missing `.env` files
- appends newly introduced variables
- preserves existing local values
- backs up modified files under `env-backup/`

## Devcontainer

This project includes a Dev Container under `.devcontainer/`.

The devcontainer provides:

- Python 3.12
- Node.js LTS
- pnpm through Corepack
- uv
- Docker-in-Docker
- forwarded ports for Web, API, Postgres, and Redis

Forwarded ports:

- `3000`: Next.js
- `5001`: FastAPI
- `5432`: Postgres
- `6379`: Redis

### Use With VS Code

Prerequisites:

- Docker Desktop
- VS Code
- Dev Containers extension

Steps:

1. Open this folder in VS Code.
2. Run `Dev Containers: Reopen in Container`.
3. Wait for the container to build.
4. The `postCreateCommand` runs `make dev-setup` automatically.
5. Start middleware and app services:

```bash
make docker-up
make dev
```

Aliases are also added inside the container:

```bash
start-api
start-web
start-stack
```

### Use With JetBrains

Prerequisites:

- Docker Desktop
- JetBrains IDE with Dev Containers support

Steps:

1. Open the project from the IDE.
2. Choose the `.devcontainer/devcontainer.json` configuration.
3. Build and open the container.
4. Run:

```bash
make docker-up
make dev
```

### Devcontainer Notes

`postCreateCommand` installs uv, enables Corepack, and runs:

```bash
make dev-setup
```

`postStartCommand` enables Corepack again to keep pnpm available after container restarts.

Docker-in-Docker is enabled so the container can run:

```bash
make docker-up
make compose-up
docker compose ps
```
